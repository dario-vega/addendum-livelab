## Using hierarchy 

### Parent table


```
CREATE TABLE IF NOT EXISTS stream_acct_hierarchy (
  id INTEGER GENERATED BY DEFAULT AS IDENTITY
    (START WITH 4 INCREMENT BY 1 NO CYCLE),

  country   STRING,
  firstName STRING,
  lastName  STRING,

  PRIMARY KEY(id)
)
```

### Child table: use showId as the key 

```
CREATE TABLE IF NOT EXISTS stream_acct_hierarchy.shows (
  showId INTEGER,
  showName STRING,
  type STRING,
  numSeasons INTEGER,
  genres ARRAY(STRING),
  seriesInfo JSON,
  PRIMARY KEY(showId)
);
```

### QUERIES 

```
SELECT * 
FROM NESTED TABLES 
(stream_acct_hierarchy a descendants(stream_acct_hierarchy.shows b))
```
